Software Life Cycle: Development phase

 a process used by the software industry to design, develop and test high quality softwares

Requirements Analysis

the process of determining user expectations for a new or modified product

Design

the process by which an agent creates a specification of a software artifact intended to accomplish goals

Implementation

Installing a new hardware or software system or application

Testing

THe process of evaluating and verifying that a software product or application does what it is supposed to do

Modular Implementation

software design technique that emphasizes separating the functionality of a program into independent ,interchangeable modules

Unified Modeling Language

a general-purpose, developmental modeling language in the field of software engineering that is intended to provide a standard way to visualize the design of a system

Use Case

a methodology used in system analysis to identify, clarify and organize system requirements

Use Case Diagram

describe the high-level functions and scope of a system

Data Realtionships:

 the process used to structure how data is stored, as well as modelling relationships within the data

one-to-one

A function is one-to-one if distinct input values are mapped by to distinct output values

one-to-many

In systems analysis, a one-to-many relationship is a type of cardinality that refers to the relationship between two entities (see also entity–relationship model) A and B in which an element of A may be linked to many elements of B, but a member of B is linked to only one element of A

many-to-many

In systems analysis, a many-to-many relationship is a type of cardinality that refers to the relationship between two entities, say, A and B, where A may contain a parent instance for which there are many children in B and vice versa. For example, think of A as Authors, and B as Books

Design Pattern

In software engineering, a software design pattern is a general, reusable solution to a commonly occurring problem within a given context in software design

Software Quality Assurance (SQA)

Software quality assurance is a means and practice of monitoring all software engineering processes, methods, and work products to ensure compliance against defined standards

Software Testing:

Software testing is the act of examining the artifacts and the behavior of the software under test by validation and verification. Software testing can also provide an objective, independent view of the software to allow the business to appreciate and understand the risks of software implementation

glass box testing

 a testing technique that examines the program structure and derives test data from the program logic/code

black box testing

Black-box testing is a method of software testing that examines the functionality of an application without peering into its internal structures or workings

beta testing

the second phase of software testing in which a sampling of the intended audience tries the product out

alpha testing

 the initial phase of validating whether a new product will perform as expected

Documentation:

User Documentation

to the documentation for a product or service provided to the end users

System Documentation

documents that describe the system itself and its parts

Technical Documentation

Technical documentation is a generic term for the classes of information created to describe the use, functionality or architecture of a product, system or service

Intellectual Property Law

deals with laws to protect and enforce rights of the creators and owners of inventions, writing, music, designs and other works, known as the "intellectual property."

Software License

a document that states the rights of the developer and user of a piece of software

Software Disclaimer

software developer's formal notice to the user of its software that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties
