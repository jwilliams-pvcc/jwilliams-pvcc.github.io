All About GitHub

  • Q. What is a version control system?
  • A. a class of systems responsible for managing changes to computer progreams, documents, large web sites, or other collections of information
  • Q. What is a Git?
  • A. Git is a free open source software for distrubuted version control: tracking changes in any set of files, usually used for coordinating work amoung programmers
  • Q. What is GitHub?
  • A.AN internet hosting service for software development and version control of Git
  • Q. What is the difference between Git and Github
  • A. Git is a version control system that lets you manage and keep track of your source code history. GitHub is a cloud-based hosting service that lets you manage Git repositories
  • Q. Who started Github and how was it started?
  • A. Chris Wanstrath, P.J. Hyett, Tom Preston-Werner, and Scott Chacon
  • Q. What company owns it now?
  • A. Microsoft
  • Q. How much does a Github account cost?
  • A.It is free

Git/Hub Terms:
  • Repository- Contains all of your projects files and each file's review history
  • Commit- an individual change to a file (or set of files)
  • Fork= a copy of a repository the you manage
  • Push- used to upload local repository content to a remote repository
  • Pull requests- Let you tell others about changes you have pushed to a branch repository on GitHub
  • workflows- a configurable automated process that will run one or more jobs
  • issues- let you track your work on GitHub, where development happens
  • raw button- open the file in a raw form, meaning that any HTML formatting disapears
  • blame button- used to examine the contents of a file line by line and see when each line was last modified and who the authr of the modification was
